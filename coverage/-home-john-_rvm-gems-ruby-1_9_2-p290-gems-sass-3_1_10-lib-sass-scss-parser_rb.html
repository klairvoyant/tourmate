405       # This is a nasty hack, and the only place in the parser
 
406       # that requires backtracking.
 
407       # The reason is that we can't figure out if certain strings
 
408       # are declarations or rulesets with fixed finite lookahead.
 
409       # For example, "foo:bar baz baz baz..." could be either a property
 
410       # or a selector.
 
411       #
 
412       # To handle this, we simply check if it works as a property
 
413       # (which is the most common case)
 
414       # and, if it doesn't, try it as a ruleset.
 
415       #
 
416       # We could eke some more efficiency out of this
 
417       # by handling some easy cases (first token isn't an identifier,
 
418       # no colon after the identifier, whitespace after the colon),

916         was = scanner.rest.dup
 
917         # Get rid of whitespace between pos and the next token,
 
918         # but only if there's a newline in there
