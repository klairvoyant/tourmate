6   # There's also a convenience method for rendering sub templates within the current controller that depends on a
 
7   # single object (we call this kind of sub templates for partials). It relies on the fact that partials should
 
8   # follow the naming convention of being prefixed with an underscore -- as to separate them from regular
 
9   # templates that could be rendered on their own.
 
10   #
 
11   # In a template for Advertiser#account:
 
12   #
 
13   #  <%= render :partial => "account" %>
 
14   #
 
15   # This would render "advertiser/_account.erb" and pass the instance variable @account in as a local variable
 
16   # +account+ to the template for display.
 
17   #
 
18   # In another template for Advertiser#buy, we could have:
 
19   #
 
20   #   <%= render :partial => "account", :locals => { :account => @buyer } %>
 
21   #
 
22   #   <% for ad in @advertisements %>
 
23   #     <%= render :partial => "ad", :locals => { :ad => ad } %>
 
24   #   <% end %>
 
25   #
 
26   # This would first render "advertiser/_account.erb" with @buyer passed in as the local variable +account+, then
 
27   # render "advertiser/_ad.erb" and pass the local variable +ad+ to the template for display.
 
28   #
 
29   # == The :as and :object options
 
30   #
 
31   # By default PartialRenderer uses the template name for the local name of the object passed into the template.
 
32   # These examples are effectively the same:
 
33   #
 
34   #   <%= render :partial => "contract", :locals => { :contract  => @contract } %>
 
35   #
 
36   #   <%= render :partial => "contract" %>
 
37   #
 
38   # By specifying the :as option we can change the way the local variable is namedin the template.
 
39   # These examples are effectively the same:
 
40   #
 
41   #   <%= render :partial => "contract", :as => :agreement
 
42   #
 
43   #   <%= render :partial => "contract", :locals => { :agreement => @contract }
 
44   #
 
45   # The :object option can be used to directly specify which object is rendered into the partial.
 
46   #
 
47   # Revisiting a previous example we could have written this code.
 
48   #
 
49   #   <%= render :partial => "account", :object => @buyer %>
 
50   #
 
51   #   <% for ad in @advertisements %>
 
52   #     <%= render :partial => "ad", :object => ad %>
 
53   #   <% end %>
 
54   #
 
55   # The :object and :as options can be used together. We might have a partial which we have named genericly,
 
56   # such as 'form'. Using :object and :as together helps us.
 
57   #
 
58   #   <%= render :partial => "form", :object => @contract, :as => :contract %>
 
59   # == Rendering a collection of partials
 
60   #
 
61   # The example of partial use describes a familiar pattern where a template needs to iterate over an array and
 
62   # render a sub template for each of the elements. This pattern has been implemented as a single method that
 
63   # accepts an array and renders a partial by the same name as the elements contained within. So the three-lined
 
64   # example in "Using partials" can be rewritten with a single line:
 
65   #
 
66   #   <%= render :partial => "ad", :collection => @advertisements %>
 
67   #
 
68   # This will render "advertiser/_ad.erb" and pass the local variable +ad+ to the template for display. An
 
69   # iteration counter will automatically be made available to the template with a name of the form
 
70   # +partial_name_counter+. In the case of the example above, the template would be fed +ad_counter+.
 
71   #
 
72   # The :as option may be used when rendering partials.
 
73   #
 
74   # Also, you can specify a partial which will be render as a spacer between each element by passing partial name to
 
75   # +:spacer_template+. The following example will render "advertiser/_ad_divider.erb" between each ad partial.
 
76   #
 
77   #   <%= render :partial => "ad", :collection => @advertisements, :spacer_template => "ad_divider" %>
 
78   #
 
79   # NOTE: Due to backwards compatibility concerns, the collection can't be one of hashes. Normally you'd also
 
80   # just keep domain objects, like Active Records, in there.
 
81   #
 
82   # == Rendering shared partials
 
83   #
 
84   # Two controllers can share a set of partials and render them like this:
 
85   #
 
86   #   <%= render :partial => "advertisement/ad", :locals => { :ad => @advertisement } %>
 
87   #
 
88   # This will render the partial "advertisement/_ad.erb" regardless of which controller this is being called from.

122   # == Rendering partials with layouts
 
123   #
 
124   # Partials can have their own layouts applied to them. These layouts are different than the ones that are
 
125   # specified globally for the entire action, but they work in a similar fashion. Imagine a list with two types
 
126   # of users:
 
127   #
 
128   #   <%# app/views/users/index.html.erb &>
 
129   #   Here's the administrator:
 
130   #   <%= render :partial => "user", :layout => "administrator", :locals => { :user => administrator } %>
 
131   #
 
132   #   Here's the editor:
 
133   #   <%= render :partial => "user", :layout => "editor", :locals => { :user => editor } %>
 
134   #
 
135   #   <%# app/views/users/_user.html.erb &>
 
136   #   Name: <%= user.name %>
 
137   #
 
138   #   <%# app/views/users/_administrator.html.erb &>
 
139   #   <div id="administrator">
 
140   #     Budget: $<%= user.budget %>
 
141   #     <%= yield %>
 
142   #   </div>
 
143   #
 
144   #   <%# app/views/users/_editor.html.erb &>
 
145   #   <div id="editor">
 
146   #     Deadline: <%= user.deadline %>
 
147   #     <%= yield %>
 
148   #   </div>
 
149   #
 
150   # ...this will return:
 
151   #
 
152   #   Here's the administrator:
 
153   #   <div id="administrator">
 
154   #     Budget: $<%= user.budget %>
 
155   #     Name: <%= user.name %>
 
156   #   </div>
 
157   #
 
158   #   Here's the editor:
 
159   #   <div id="editor">
 
160   #     Deadline: <%= user.deadline %>
 
161   #     Name: <%= user.name %>
 
162   #   </div>
 
163   #
 
164   # You can also apply a layout to a block within any template:
 
165   #
 
166   #   <%# app/views/users/_chief.html.erb &>
 
167   #   <%= render(:layout => "administrator", :locals => { :user => chief }) do %>
 
168   #     Title: <%= chief.title %>
 
169   #   <% end %>
 
170   #
 
171   # ...this will return:
 
172   #
 
173   #   <div id="administrator">
 
174   #     Budget: $<%= user.budget %>
 
175   #     Title: <%= chief.name %>
 
176   #   </div>
 
177   #
 
178   # As you can see, the <tt>:locals</tt> hash is shared between both the partial and its layout.
 
179   #
 
180   # If you pass arguments to "yield" then this will be passed to the block. One way to use this is to pass
 
181   # an array to layout and treat it as an enumerable.
 
182   #
 
183   #   <%# app/views/users/_user.html.erb &>
 
184   #   <div class="user">
 
185   #     Budget: $<%= user.budget %>
 
186   #     <%= yield user %>
 
187   #   </div>
 
188   #
 
189   #   <%# app/views/users/index.html.erb &>
 
190   #   <%= render :layout => @users do |user| %>
 
191   #     Title: <%= user.title %>
 
192   #   <% end %>
 
193   #
 
194   # This will render the layout for each user and yield to the block, passing the user, each time.
 
195   #
 
196   # You can also yield multiple times in one layout and use block arguments to differentiate the sections.
 
197   #
 
198   #   <%# app/views/users/_user.html.erb &>
 
199   #   <div class="user">
 
200   #     <%= yield user, :header %>
 
201   #     Budget: $<%= user.budget %>
 
202   #     <%= yield user, :footer %>
 
203   #   </div>
 
204   #
 
205   #   <%# app/views/users/index.html.erb &>
 
206   #   <%= render :layout => @users do |user, section| %>
 
207   #     <%- case section when :header -%>
 
208   #       Title: <%= user.title %>
 
209   #     <%- when :footer -%>
 
210   #       Deadline: <%= user.deadline %>
 
211   #     <%- end -%>
 
212   #   <% end %>
